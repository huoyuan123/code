# 时间复杂度的计算方法

## 一、怎么计算时间复杂度（Big-O）

- 明确输入规模 n
	- 数组/字符串长度 L、矩阵规模 m×n、数字大小的位数 log n、递归层数等。
- 选定“基本操作”
	- 例如一次比较、一次赋值、一次入栈/出栈等。
- 计数并写出表达式
	- 顺序语句相加，循环语句按循环次数相乘，嵌套循环乘法叠加。
- 取最坏情况（通常默认）
	- 有需要时也可分析平均/最好情况。
- 忽略常数与低阶项
	- 只保留最高阶量级，得到 O(·)。

## T(n) 是什么（运行时间函数）

- 定义：T(n) 表示输入规模为 n 时，算法执行所需“工作量”的函数，常用“基本操作次数”来度量（与具体机器/语言无关）。
- 成本模型：选择一种基本操作作为计数单位（如比较、赋值、入栈/出栈）。例如 for 循环体是常数开销 c，循环 n 次 ⇒ T(n)≈c·n+b。
- 情况区分：
	- 最坏时间 T_worst(n)：在所有输入中用时最长（默认分析对象）。
	- 平均时间 T_avg(n)：在随机输入上的期望用时。
	- 最好时间 T_best(n)：在最有利输入上的最短用时。
- 与大 O 的关系：用 O/Θ/Ω 来描述 T(n) 的上界/紧确界/下界。例如 T(n)=3n+5：
	- 上界 O(n)，紧确 Θ(n)，下界 Ω(n)。
- 小例子：
	- 单层循环常数体：T(n)=an+b ⇒ O(n)
	- 双层 n×n 常数体：T(n)=a·n^2+… ⇒ O(n^2)
	- 指数级回溯：T(n)≈c·2^n ⇒ O(2^n)

## 附：O(f(n)) 表示什么（澄清“括号里的内容”）

- 形式化定义：若存在常数 c>0、n0，使得对所有 n≥n0，T(n) ≤ c·f(n)，则 T(n) ∈ O(f(n))。
- 含义：用增长函数 f(n) 描述算法随 n 变大时的“上界量级”。
- 常见写法：
	- O(1)：常数级（与 n 无关，不等于“恰好 1 次”）
	- O(log n)：对数级（log 底数差异只差常数倍，在大 O 中可忽略）
	- O(n)、O(n log n)、O(n^2)、O(n^k)：线性/线性对数/多项式级
	- O(2^n)、O(n!)：指数/阶乘级（常见于回溯/穷举）
- 为什么可以忽略常数与低阶项：
	- 例：T(n)=3n+5 ⇒ O(n)；T(n)=n^2+100n+7 ⇒ O(n^2)。当 n 很大时主导项决定增长。
- 易混点：
	- O(1) 不是“1 次操作”，而是“与 n 无关的常量级”。
	- O(·) 是上界；还有 Θ(·)（紧确界）、Ω(·)（下界）。常用 O(·) 表示最坏上界。

小口诀：总复杂度 ≈ 迭代次数 × 每次代价；嵌套循环相乘，分治看 Master，回溯看“分支数^深度”。

## 二、常见结构的快判规则

- 顺序执行：T(n)=T1(n)+T2(n) ⇒ O(max{T1,T2})
- for 循环：循环次数 × 循环体开销
	- for(i=0；i<n；i++) O(1) ⇒ O(n)
	- 双重嵌套 n×n ⇒ O(n^2)
- while 循环（按增长/减半规律）
	- i*=2（每次翻倍到达 n）⇒ 约 log2 n 次 ⇒ O(log n)
	- i--（每次减 1 到 0）⇒ 约 n 次 ⇒ O(n)
- 分治递归 T(n)=aT(n/b)+f(n)
	- 典型：归并/快排 O(n log n)
	- Master 定理（简表）：
		- f(n)=O(n^k)，若 a < b^k ⇒ O(n^k)
		- 若 a = b^k ⇒ O(n^k log n)
		- 若 a > b^k ⇒ O(n^{log_b a})
- 回溯/枚举
	- 子集/组合/排列常见 O(2^n), O(n·2^n), O(n!) 等（取决于分支数和深度）。
- 栈/队列单次操作：一般 O(1)

## 三、函数/算法复杂度举例（与本仓库常见代码呼应）

- ConvertBase_LinkStack(n,d)
	- while(num>0){num/=d} 循环次数等于位数 ≈ log_d n
	- 时间：O(log n)，空间：O(log n)
- InfixToPostfix(expr)、EvalPostfix(postfix)
	- 单次线性扫描，栈操作均摊 O(1)
	- 时间：O(L)（L 为表达式长度），空间：O(L)
- 迷宫 DFS（MazeSolve）
	- 每个格子最多访问/回溯常数次
	- 时间：O(m·n)，空间：O(m·n)（visited + 路径栈）
- 汉诺塔 hanoi(n)
	- 递归 T(n)=2T(n−1)+O(1)
	- 时间：O(2^n)，空间：O(n)（递归深度）
- Ackermann 函数
	- 增长极快，远超指数；仅对极小参数可计算
- 0/1 背包“恰好装满”的子集和递归判定
	- 每个物品“选/不选”两分支，最坏遍历 2^n 组合
	- 时间：O(2^n)，空间：O(n)
	- 若引入记忆化（i, remain）可降至 O(n·T)

## 四、示例推导

- while(num>0){ num/=d }
	- num 依次为 n, n/d, n/d^2,… 直到 <1，次数 k≈log_d n ⇒ O(log n)
- 汉诺塔 hanoi
	- T(n)=2T(n−1)+1 ⇒ 展开得 T(n)=2^n−1 ⇒ O(2^n)
- DFS 迷宫
	- 每格至多被访问并回溯一次，边界与障碍剪枝不影响上界 ⇒ O(mn)

## 五、空间复杂度（补充）

- 同法计算额外使用的存储量（不含输入本身），如递归深度、辅助数组大小等。

提示：学习与面试中，先用“快判规则”估计量级，再按需要做严格推导；对回溯/递归问题，特别关注分支数与递归深度对复杂度的指数式影响。


## 附录：函数 → 复杂度简表（与 `线性表/栈.c` 对应）

- `InitStackPos`：时间 O(1)，空间 O(1)
- `IsEmptyPos`：时间 O(1)，空间 O(1)
- `PushPos`：时间 O(1)，空间 O(1)
- `PopPos`：时间 O(1)，空间 O(1)
- `InitLinkStack`：时间 O(1)，空间 O(1)
- `PushLink`：时间 O(1)，空间 O(1)
- `PopLink`：时间 O(1)，空间 O(1)
- `ConvertBase_LinkStack`：时间 O(log n)，空间 O(log n)（n 为十进制数大小；复杂度按位数计）
- `dfs_maze`：时间 O(m·n)，空间 O(m·n)
- `MazeSolve`：时间 O(m·n)，空间 O(m·n)
- `Priority`：时间 O(1)，空间 O(1)
- `InfixToPostfix`：时间 O(L)，空间 O(L)（L 为表达式长度）
- `EvalPostfix`：时间 O(L)，空间 O(L)
- `TestExpr`：时间 O(L)，空间 O(L)
- `hanoi`：时间 O(2^n)，空间 O(n)
- `ackermann`：时间超指数（远超 2^n），空间取决于递归深度
- `subset_sum_dfs`：时间 O(2^n)，空间 O(n)
- `Knapsack01_MaxValue`：时间 O(2^n)，空间 O(n)

注：L 表达式长度；m、n 为迷宫行列；背包/子集和的 n 指物品个数。若对子集和/背包加入记忆化 (i, remain) 备忘，则时间可降至 O(n·T)，空间 O(n·T)。
